Änderungen mit Magento 2.4.4

Patchnotes 2.4.4:

"Email variable usage was deprecated back in 2.3.4 as part of a security risk mitigation in favor of a more strict variable syntax. This legacy behavior has been fully removed in this release as a continuation of that security risk mitigation."

Was wirklich passiert

## Technik `LegacyResolver` & `StrategyResolver` werden entfernt. D.h. ein Fallback auf `strict mode` Damit stehen keine direkten Methoden Aufrufe mehr zur Verfügung sondern nur noch einfache Werte.

Wie bitte?

Früher: {{var subscriber.getConfirmationLink()}}
Jetzt: {{var subscriber_data.confirmation_link}}
Bitte darauf achten, keine Leerzeichen an den
geschweiften Klammern '{{}}' zu verwenden:
Richtig: {{var subscriber_data.confirmation_link}}
Falsch: {{ var subscriber_data.confirmation_link }}

Todos

  • Base Templates sind angepasst worden.
  • Custom Templates (DB) müssen verändert werden.
  • Custom Templates (Theme) müssen verändert werden.
Sonst kann es passieren das die fehlerhaften Emails nicht mehr versandt werden!
